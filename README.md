Terraform Next.js Portfolio Deployment on AWS

This project deploys a static Next.js portfolio website to AWS using Terraform. The infrastructure uses a private S3 bucket for static site files and Amazon CloudFront as the public CDN layer. CloudFront accesses S3 securely through Origin Access Control, keeping the bucket private while serving the website globally over HTTPS.

Project Overview

The goal of this project was to build a production-style static website deployment using Infrastructure as Code. Instead of manually creating AWS resources through the console, the infrastructure is defined with Terraform and deployed through repeatable commands.

This project demonstrates:

- Static Next.js site generation
- AWS S3 private bucket hosting
- Amazon CloudFront CDN distribution
- CloudFront Origin Access Control
- S3 bucket policy restricted to CloudFront
- Terraform-managed infrastructure
- AWS CLI deployment workflow
- Git and GitHub version control

Amazon S3

S3 stores the static files generated by the Next.js build process. The bucket is private and blocks public access.

Amazon CloudFront

CloudFront serves the website globally through a CDN, improving performance and providing HTTPS access.

CloudFront Origin Access Control

Origin Access Control allows CloudFront to securely retrieve objects from the private S3 bucket without exposing the bucket publicly.

IAM / Bucket Policy

The S3 bucket policy only allows the CloudFront distribution to read objects from the bucket.

Terraform

Terraform provisions and manages the AWS infrastructure, including:

- S3 bucket
- S3 public access block

Security Design

This project avoids using a public S3 website endpoint. Instead, the S3 bucket remains private and CloudFront is the only public entry point.

Security choices include:

- S3 public access blocked
- No public bucket policy
- CloudFront Origin Access Control enabled
- S3 bucket policy restricted to the CloudFront distribution ARN
- HTTPS enforced through CloudFront viewer protocol policy
- Terraform state and variable files excluded from GitHub

Deployment Steps

1. Install dependencies
   cd blog
   npm install
2. Build and export the Next.js site
   npm run build
   The static files are generated in:The static files are generated in:
3. Deploy infrastructure with Terraform
   cd ../terraform
   terraform init
   terraform validate
   terraform plan
   terraform apply
4. Upload static files to S3
   From inside the terraform/ directory:
   aws s3 sync ../blog/out s3://ladale-nextjs-portfolio-2026 --delete
5. View Terraform outputs
   terraform output
   Example output:
   bucket_name = "ladale-nextjs-portfolio-2026"
   cloudfront_domain_name = "d1hsikif338cit.cloudfront.net"
